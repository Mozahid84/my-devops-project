🏢 Ansible + AWX Implementation Guide

✓ Enterprise Solution - Team collaboration, RBAC, audit logs, and CI/CD integration

Overview

The Ansible approach provides a complete Infrastructure-as-Code solution for enterprise environments. It's perfect for teams that need:

  • Team-based orchestration
  • Role-based access control (RBAC)
  • Audit trails and logging
  • Complex deployment workflows
  • GitLab CI/CD integration
  • Version control

Key Components

📋 Playbooks

site.yml - Main deployment playbook
backup.yml - Backup & restore playbook

🔧 Roles

mssql - Complete MSSQL installation and configuration role

📝 Inventory

Hosts, groups, and variables for target VMs

🎛️ AWX

Optional orchestration UI for team management

Quick Start (5 Minutes)

Step 1: Navigate to Project

cd ansible-mssql-deploy

Step 2: Update Inventory

# Edit inventory/hosts.ini with your VM IPs

# Edit group_vars/mssql_servers.yml with strong SA password

Advanced Commands

Dry Run (Test Without Making Changes)

ansible-playbook -i inventory/hosts.ini playbooks/site.yml -C

Run Specific Tasks

ansible-playbook -i inventory/hosts.ini playbooks/site.yml -t install
ansible-playbook -i inventory/hosts.ini playbooks/site.yml -t configure
ansible-playbook -i inventory/hosts.ini playbooks/site.yml -t adventureworks

Run on Specific Host

ansible-playbook -i inventory/hosts.ini playbooks/site.yml -l vm1
ansible-playbook -i inventory/hosts.ini playbooks/site.yml -l vm2

Verbose Output (Debugging)

ansible-playbook -i inventory/hosts.ini playbooks/site.yml -vvv

Backup Only (VM1)

ansible-playbook -i inventory/hosts.ini playbooks/backup.yml -l vm1

Restore Only (VM2)

ansible-playbook -i inventory/hosts.ini playbooks/backup.yml -l vm2

Task Breakdown

Phase 1: Installation (10-15 minutes)

  • Install system dependencies
  • Add Microsoft repositories
  • Install MSSQL Server package
  • Install MSSQL Tools
  • Run MSSQL setup
  • Start SQL Server service

Phase 2: Configuration (5-10 minutes)

  • Create backup directories
  • Create data directories
  • Set default database paths
  • Configure network settings
  • Enable SQL Agent

Phase 3: Database Restore (10-15 minutes)

  • Download AdventureWorks backup
  • Restore database on both VMs
  • Verify database integrity

Phase 4: Backup Creation (10-20 minutes, VM1 only)

  • Create backup directory
  • Create 10-stripe backup
  • Verify backup files

Phase 5: Restore (10-20 minutes, VM2 only)

  • Fetch backups from VM1
  • Copy to local directory
  • Transfer to VM2
  • Restore from striped backup
  • Verify restore success

  2. Push ansible-mssql-deploy project
  3. Add SSH_PRIVATE_KEY as CI variable
  4. Pipeline runs automatically on push

CI/CD Stages

  • Lint: Syntax check and ansible-lint
  • Deploy: Run playbook (manual trigger)

Troubleshooting

SSH Connection Issues

Problem: Permission denied (publickey)

Solution: Verify SSH keys are set up correctly

ssh-copy-id -i ~/.ssh/id_rsa root@VM_IP

Ansible Issues

Problem: Task failed with "UNREACHABLE"

Solution: Verify inventory and SSH connectivity

ansible all -i inventory/hosts.ini -m ping

MSSQL Issues

Problem: MSSQL service won't start

Solution: Check system resources and SA password requirements

ansible all -i inventory/hosts.ini -m shell -a "systemctl status mssql-server"

Performance Tips

  • Use -C flag for dry runs before actual deployment
  • Use -t flag to run specific tags for faster iteration
  • Monitor resource usage with -vvv for detailed output
  • Use -l flag to limit to specific hosts for testing
  • Backup before making configuration changes
