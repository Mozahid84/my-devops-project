📋 Complete Setup Guide

⏱️ Time: 30-40 minutes from start to first deployment

Prerequisites

Hardware Requirements

  • Control Machine: Windows 10/11 with WSL2 or Git Bash (or Linux/Mac)
  • Target VMs: Two CentOS 8 / RHEL 8 systems
  • Disk Space: 5GB+ per VM for MSSQL and backups
  • Network: All systems on same network segment
  • Memory: 2GB+ per VM (4GB+ recommended)

Software Requirements

  • Control Machine:
    • Python 3.9+
    • Ansible 2.10+
    • SSH client
    • Git (optional)
  • Target VMs:
    • CentOS 8 / RHEL 8
    • SSH server
    • sudo access (or root)

Step 1: Prepare SSH Keys (5 minutes)

1.1 Generate SSH Keys

On your control machine (Windows/Linux/Mac):

ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa -N ""

1.2 Copy to Target VMs

For each VM (vm1 at 192.168.56.101, vm2 at 192.168.56.102):

ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.56.101
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.56.102

⚠️ Ensure SSH Works

Test SSH access to each VM:

ssh -i ~/.ssh/id_rsa root@192.168.56.101
exit
ssh -i ~/.ssh/id_rsa root@192.168.56.102
exit

Step 2: Setup Ansible (10 minutes)

2.1 Install Ansible

# On Windows (WSL2/Git Bash)
python -m pip install ansible==2.10.0

# On Linux/Mac
brew install ansible  # macOS
sudo apt install ansible  # Ubuntu/Debian

2.2 Verify Installation

ansible --version

2.3 Update Inventory

Edit ansible-mssql-deploy/inventory/hosts.ini:

[mssql_servers]
vm1 ansible_host=192.168.56.101 instance_name=instance1
vm2 ansible_host=192.168.56.102 instance_name=instance2

[mssql_servers:vars]
ansible_user=root
ansible_ssh_private_key_file=~/.ssh/id_rsa
ansible_python_interpreter=/usr/bin/python3

2.4 Test Ansible Connectivity

ansible all -i ansible-mssql-deploy/inventory/hosts.ini -m ping

✅ Expected Output

vm1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
vm2 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}

Step 3: Configure Variables (10 minutes)

3.1 Update Group Variables

Edit ansible-mssql-deploy/group_vars/mssql_servers.yml:

---
mssql_version: 2019
sa_password: "YourStrongPassword123!@#"  # Change this!
mssql_port: 1433
backup_dir: /backup/striped
data_dir: /var/opt/mssql/data
backup_location: primary

3.2 Update Host Variables for VM1

Edit ansible-mssql-deploy/host_vars/vm1.yml:

---
instance_name: instance1
backup_location: primary

3.3 Update Host Variables for VM2

Edit ansible-mssql-deploy/host_vars/vm2.yml:

---
instance_name: instance2
backup_location: secondary

⚠️ Important: SA Password

Make sure to use a strong password that meets SQL Server requirements:

  • At least 8 characters
  • Mix of uppercase, lowercase, numbers, and special characters

Step 4: Choose Your Deployment Method (1 minute)

Option A: Ansible Playbook (Enterprise)

Use this for team environments with complex workflows.

cd ansible-mssql-deploy
ansible-playbook -i inventory/hosts.ini playbooks/site.yml

Option B: FastAPI REST API (Lightweight)

Use this for simple, lightweight deployments with API control.

cd python-fastapi-mssql
python -m venv venv
source venv/Scripts/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload

Step 5: Verify Prerequisites (5 minutes)

5.1 Check Disk Space on VMs

ansible all -i ansible-mssql-deploy/inventory/hosts.ini -m shell -a "df -h /"

Verify that at least 5GB is available.

5.2 Check Network Configuration

ansible all -i ansible-mssql-deploy/inventory/hosts.ini -m shell -a "hostname -I"

5.3 Test Port Availability

ansible all -i ansible-mssql-deploy/inventory/hosts.ini -m shell -a "netstat -tlnp | grep 1433 || echo 'Port 1433 is available'"

Troubleshooting

SSH Connection Issues

Problem: SSH key permission denied

Fix file permissions:

chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_rsa
chmod 644 ~/.ssh/id_rsa.pub

Problem: Host key verification failed

Add hosts to known_hosts:

ssh-keyscan -H 192.168.56.101 >> ~/.ssh/known_hosts
ssh-keyscan -H 192.168.56.102 >> ~/.ssh/known_hosts

Ansible Issues

Problem: Ansible command not found

Reinstall Ansible:

python -m pip install --upgrade ansible

Problem: Inventory file not found

Check file paths are correct and use absolute paths if needed.

MSSQL Issues

Problem: Port 1433 already in use

Change MSSQL port in group_vars or check what's using the port:

netstat -tlnp | grep 1433

Problem: SA password requirements not met

Use a strong password with uppercase, lowercase, numbers, and special characters.
