⚡ FastAPI Implementation Guide

✓ Lightweight Solution - Simple REST API with Docker support and real-time monitoring

Overview

The FastAPI approach provides a simple, lightweight REST API service for MSSQL deployment. It's perfect for teams that need:

  • Simple REST API interface
  • Easy integration with existing tools
  • Docker containerization
  • Real-time monitoring endpoints
  • Quick setup and deployment
  • Low operational overhead

Key Components

⚡ FastAPI App

main.py - Core application
config.py - Configuration

🔧 Ansible Runner

ansible_runner.py - Subprocess execution

🛣️ Routes

deploy.py - Deployment endpoints
health.py - Health checks
logs.py - Logging endpoints

🐳 Docker

Dockerfile and docker-compose.yml for containerization

Quick Start (5 Minutes)

Step 1: Navigate to Project

cd python-fastapi-mssql

Step 2: Create Virtual Environment

python -m venv venv
source venv/Scripts/activate  # Windows: venv\Scripts\activate

Step 3: Install Dependencies

pip install -r requirements.txt

Step 4: Configure Environment

cp .env.example .env
# Edit .env with your configuration

Step 5: Start Server

uvicorn app.main:app --reload

Step 6: Access API

Visit: http://localhost:8000/api/docs

API Endpoints

Root Endpoints

Method Endpoint Description
GET / Service information
GET /api/v1 API version

Deployment Endpoints (/deploy)

Method Endpoint Description
POST /deploy/install Start full MSSQL deployment
POST /deploy/backup Create 10-stripe backup on VM1
POST /deploy/install-tools Install MSSQL tools only
POST /deploy/restore-db Restore AdventureWorks database
GET /deploy/status Get deployment status
GET /deploy/hosts Get inventory hosts
GET /deploy/history Get execution history
POST /deploy/ping Test connectivity to hosts

Health Check Endpoints (/health)

Method Endpoint Description
GET /health/check Basic health status
GET /health/ready Readiness probe (all checks)
GET /health/live Liveness probe

Logging Endpoints (/logs)

Method Endpoint Description
GET /logs/latest Last N lines of logs
GET /logs/level/{level} Filter logs by level
GET /logs/since Logs from last N minutes
POST /logs/clear Clear log file
GET /logs/info Log file statistics

Docker Deployment

Using Docker Compose (Recommended)

# Start the service
docker-compose up -d

# View logs
docker-compose logs -f

# Stop the service
docker-compose down

Using Docker Directly

# Build image
docker build -t mssql-fastapi .

# Run container
docker run -d -p 8000:8000 \
  -e ANSIBLE_INVENTORY=/ansible/inventory/hosts.ini \
  -e MSSQL_SA_PASSWORD=Password123!@# \
  -v ~/.ssh:/root/.ssh:ro \
  mssql-fastapi

Background Task Execution

Deployment tasks run in the background and return immediately with a 202 Accepted response. Use the endpoints below to monitor progress:

  • GET /logs/latest - View current log output
  • GET /deploy/history - Check execution status
  • GET /health/ready - Verify system readiness

Health Checks

What Gets Checked

  • Ansible Availability: Is Ansible installed?
  • Inventory Access: Can we read the inventory file?
  • Disk Space: Is there ≥5GB available?
  • CPU Usage: Is CPU <90%?
  • Memory Usage: Is memory <90%?

Troubleshooting

Port Already in Use

Problem: Address already in use

Solution: Change port or kill existing process

uvicorn app.main:app --port 8001
# or
lsof -i :8000 | grep LISTEN | awk '{print $2}' | xargs kill -9

Ansible Not Found

Problem: Ansible command not found

Solution: Install Ansible in virtual environment

pip install ansible

Inventory File Not Found

Problem: Cannot find inventory file

Solution: Update ANSIBLE_INVENTORY environment variable

export ANSIBLE_INVENTORY=/path/to/inventory/hosts.ini

Performance Tips

  • Use /health/ready to verify system readiness before deployment
  • Monitor /logs/latest during long-running deployments
  • Check /deploy/history to see past execution results
  • Use Docker for consistent environments
  • Set appropriate timeouts for your environment
